In the example above, many animations have already been implemented using CSS, but in almost every operating system and browser, user preferences can be set to allow users to choose whether to enable animation effects. This feature is important for some people because they may feel that animation effects can be uncomfortable or distracting.

[prefers reduced motion](https://developer.mozilla.org/en-US/docs/Web/CSS/@media/prefers-reduced-motion)

Below we introduce how to check if the user has enabled `prefers-reduced-motion`. If it is enabled, we need to remove the animation effects.

## Remove logo animation when user prefers reduced motion

```css
header.nav h1 svg .cls-1 {
  fill: rgba(255, 255, 255, 1);
  transition: 0.7s;
  stroke: rgba(255, 255, 255, 0);
  stroke-width: 2px;
}
header.nav h1 svg.animate .cls-1 {
  fill: rgba(255, 255, 255, 0);
  stroke: var(--green);
  stroke-dasharray: 1000;
  animation: 3s ease-in-out drawLogo;
}

@keyframes drawLogo {
  from {
    stroke-dashoffset: 1000;
  }
  to {
    stroke-dashoffset: 0;
  }
}
```

Above code animates the logo stroke, we can disable this animation when the user prefers reduced motion by using the `prefers-reduced-motion` media query as follows:

```css
@media(prefers-reduced-motion) {
  header.nav h1 svg.animate .cls-1 {
    animation: none;
    fill: rgba(255, 255, 255, 1);
    stroke: rgba(255, 255, 255, 0);
  }
}
```

    }, 5000);
  });

  // check if user prefers reduced motion, if so, pause all videos, if not, play all videos
  const mediaQuery = matchMedia("(prefers-reduced-motion)")
  const checkReducedMotion = () => {
    const videos = document.querySelectorAll(".bg-video");
    // pause or play videos based on user preference `prefers-reduced-motion`
    if (mediaQuery.matches) {
      videos.forEach(v => v.pause());
    } else {
      videos.forEach(v => v.play());
    }
  };
  checkReducedMotion(); // initial check on page load
  mediaQuery.addEventListener("change", checkReducedMotion); // if user changes preference, update videos accordingly
});
